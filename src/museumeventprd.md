# ArtPulse (艺脉) 核心功能技术规格说明

## 1. 城市看展力引擎 (City Art Index, CAI)
**目标：** 通过量化指标告诉用户“这周末去哪”。

### 业务逻辑
系统需对每个城市实时计算一个 **CAI 分值**：
$$CAI = (E_{ongoing} \times 0.6) + (E_{ending} \times 1.5) + (R_{avg} \times 2.0)$$
* **$E_{ongoing}$**: 当前正在进行的展览数量。
* **$E_{ending}$**: 7天内即将结束的展览数量（权重最高，体现紧迫性）。
* **$R_{avg}$**: 该城市所有展览的平均评分。

### 展示方案 (UI/UX)
* **热力图模式：** Web端首页使用 WebGL 渲染的全球/全国底图，城市节点光圈大小随 CAI 分值闪烁。
* **城市排行榜：** 移动端顶部滑块，点击城市名弹出“看展简报”（如：“上海：本周 3 个新展，5 个闭幕，最值得去”）。

## 2. “生命周期”时间轴列表 (Lifespan Timeline)
**目标：** 解决用户对展期长短的感知模糊问题。

### 开发需求
* **组件设计：** 每一个展览卡片内置一个 `Exhibition-Progress-Bar`。
* **计算公式：**
    * `Progress = (Today - start_date) / (end_date - start_date)`
* **视觉分态：**
    1.  **准备期 (Progress < 0):** 进度条为虚线，标记“期待中”。
    2.  **热展期 (0 < Progress < 0.8):** 进度条为品牌色（如克莱因蓝）。
    3.  **抢救期 (Progress > 0.8):** 进度条变为亮红色，并伴有“Last Call”闪烁标签。
    4.  **已结束 (Progress > 1.0):** 卡片整体 40% 透明度，置灰。

## 3. AI 结构化详情页 (AI-Driven Details)
**目标：** 极致的阅读效率，过滤废话。

### 核心组件
1.  **沉浸式 Header：**
    * 使用 `poster_url` 作为背景，采用 `Canvas` 提取图片主色调作为页面强调色。
2.  **Highlight 标签云：**
    * 将 `highlights` 数组通过词云或卡片形式平铺。**开发注意：** 移动端支持左右滑动切换亮点。
3.  **智能开馆状态机：**
    * **后端逻辑：** 解析 `open_hours` 字符串。需识别关键词（如“周一闭馆”、“延长至21:30”）。
    * **前端展示：** 实时显示状态标签：`● 开放中`、`○ 闭馆中`、`⏰ 即将闭馆`。

### 交互原型
* **点击地址：** 调起第三方地图（Google/高德/百度）并自动填入 `address`。
* **收藏动作：** 点击收藏后，该展览自动加入用户的“待办时间轴”。

## 4. 智能观展路径规划 (Smart Itinerary Planner)
**目标：** 解决多展连看的交通与时间冲突。

### 算法需求：TSP (旅行商问题) 简化版
当用户选中 3 个以上的展览并点击“一键排程”：
* **输入：** 多个 `address` 点位、各馆 `open_hours`、用户当前位置。
* **处理逻辑：**
    1.  计算点位间的欧几里得距离或通过地图 API 获取通行时间。
    2.  **冲突校验：** 若 A 展 17:00 闭馆，B 展 21:30 闭馆，算法必须优先将 A 展排在前面。
    3.  **午休建议：** 在两展之间若超过 2 小时空隙，自动插入“附近高分咖啡馆/餐厅”。

### 界面展示
* **生成路线图：** 类似地铁线路图的简约 UI。
* **时间刻度线：** 垂直排列：09:00 出发 -> 10:30 展览A -> 13:00 简餐 -> 15:00 展览B。

---

## 6. 非功能性需求 (Engineering Tips)

* **图片懒加载：** 由于展览海报（`poster_url`）质量极高，需进行 WebP 压缩转换，防止页面卡顿。
* **离线存储：** 使用 `IndexedDB` 存储用户已规划的行程，确保在美术馆地下层等信号差的地方也能查看路线。
* **响应式适配：**
    * **Mobile:** 侧重于“找展”和“导航”。
    * **Web:** 侧重于“深度阅读”和“长途旅行规划”。
